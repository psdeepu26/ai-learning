🤖 How LLMs Work

Day #X · AI Learning — Like you're 5, then like you're an engineer
🧒 LIKE YOU'RE 5
Imagine you have a magic word machine. You type "Once upon a..." and the machine guesses the next word. It thinks:

🤔 "Hmm, people usually say 'Once upon a time'. But also sometimes 'Once upon a story'. I'll pick 'time' — that feels safest."

The machine does this one word at a time, always guessing the next word based on everything you've said so far. It's like playing that game where you build a story one word at a time — except this machine read almost everything on the internet to practice, so it's really good at guessing.

The secret: It doesn't "think" or "understand" like you do. It just got really, really good at the guessing game.
↓ Now let's call it what it really is ↓
🔧 FOR THE ENGINEER
🔤

Tokenization

"Hello world" → ["Hello", " wo", "rld"]. ~0.75 tokens per word. DeepSeek V4 handles 128K tokens per request (~170K words — a whole novel).

🧠

Self-Attention

Every token scores its relevance to every other token. The Q·K^T / √d matrix determines "how much should I pay attention to word #5 when generating word #12?"

🎲

Temperature

Controls the probability distribution over next tokens. temp=0 → greedy (always most likely). temp=0.7 → creative. top_p=0.9 → nucleus sampling (only consider top 90% probability mass).

🏗️

MoE Architecture

DeepSeek V4 uses Mixture of Experts: 680B total params, but only ~37B active per token. A router network picks which "expert" sub-networks to activate — like having 680 specialists but paying only 37.

📚

3 Training Stages

1. Pretraining (predict next word on internet text) → 2. Instruction fine-tuning (Q&A format) → 3. RLHF (human preferences via reward model).

🔗

RAG (Retrieval-Augmented Gen)

Before generating, search a vector DB for relevant documents. Inject them into context. No retraining needed. This is how Hermes knows your config.yaml without memorizing it.
